## 🎯 Key Directories Reference

### When you need...

**To add API calls**
→ Look in `src/api/` - See examples in `authAPI.ts`

**To use auth**
→ Import `useAuth` from `src/context/AuthContext.tsx`

**To create a UI element**
→ Check `src/components/ui/` for existing components

**To add a new page**
→ Create file in `src/pages/` and add route in `src/app/router.tsx`

**To add a feature**
→ Create folder in `src/features/` with its own API module

**To see styling examples**
→ Check existing components like `src/components/ui/Button.tsx`

**To understand the structure**
→ Read `ARCHITECTURE.md`

**Quick usage patterns**
→ See `QUICK_REFERENCE.md`

**Migration from old code**
→ Check `MIGRATION_GUIDE.md`

## 💾 Important: Files to Commit to Git

✅ Commit these files:
- All `src/` files
- Config files: `vite.config.ts`, `tsconfig.json`, etc.
- `.gitignore`, `.env.example`
- Documentation: `ARCHITECTURE.md`, `MIGRATION_GUIDE.md`, etc.
- `package.json`, `package-lock.json`

❌ Do NOT commit:
- `node_modules/` (generated by `npm install`)
- `.env` (has secrets - create from `.env.example`)
- `dist/` (generated by `npm run build`)
- `.DS_Store`, `*.log` files (ignored by `.gitignore`)

## 🔄 How Files Reference Each Other

**Example dependency chain:**
```
pages/Dashboard.tsx
  ├── imports PageWrapper from components/layout
  ├── imports DashboardStats from features/dashboard
  │   └── (DashboardStats may use Button from components/ui)
  └── wrapped by AuthProvider from context

app/router.tsx
  └── imports all pages and wraps protected ones in ProtectedRoute

App.tsx
  ├── uses AuthProvider from context
  └── uses AppRouter from app/router.tsx

main.tsx
  └── imports App.tsx and renders to #root in index.html
```
